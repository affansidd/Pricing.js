Pricing.js API Documentation

Examples Home

Welcome to the official API documentation for the Pricing.js library. Here you will find information about how to get set-up with the library and details about all the functionality that the library provides.

Getting Started:

To get started with the Pricing.js library, first download the libary code from here. Ensure the icons folder is in the same directory as the js folder.
Next, include the following tags in your html file to include library functionality in your web app:

<script type="text/javascript" src=".../pub/js/pricing.js"></script>

<link rel="stylesheet" type="text/css" href=".../pub/pricing.css">

Once the pricing.js and pricing.css files have been linked, you're ready to start using the Pricing.js library to create unique and intuitive pricing matrices to display your products and/or services!

The following code snippet demonstrates how to create a pyramid pricing matrix to represent a tiered service structure. First create a JS object that represents the product information, according to the specifications of the type of matrix you wish to creatte. Then use the constructor for the type of matrix you want to build to create an instance of a generator. Finally, use the generators make function to create the object and add it to a container of your choice. 

                    
            
    //create an object that represents the product information (more details about the requirements for each type of matrix can be found below)

    const subscriptionPyramid = [
        {
            value: "Prestige",
            text: "Price: $30. Our most prestige package offers unlimited usage and our top speeds that will never slow down",
            color: '#d766ed'
        }, 
        {
            value: "Intermediate: $25",
            text: "The intermediate package offers 500GB monthly usage of high speed internet. (Speed may slow down during peak times)",
            color: '#eda166'
        }, 
        {
            value: "Basic: $15",
            text: "The basic package offers 100GB monthly usage of moderate speed internet. This package is suitable for those with minimal internet needs.",
            color: '#DAF7A6'
        }
    ]

    const container = document.querySelector('body')

    const pyramidGenerator = new PyramidGenerator()
    const pyramidNum = pyramidGenerator.makePyramid(subscriptionPyramid, container)

The same process applies for each of the different types of matrices the library can create. Simply create an object according to the specifications of the type of matrix you want to create (which can be found below), create (if you haven't already) an object using the constructor of the desired matrix and use the the make function to generate the pricing matrix.

Matrix Generators:

Every Generator object has two attributes, an array that stores an object representing each matrix that this generator has been used to create and the number of matrices this generator has created. Each object in the array is kept up to date with any dynamic changes that are made to a matrix using the API functions of a generator. A developer can access this array at any time to see which trees have been created and the current state of a matrix via the info field of each object.

Tree Generator:

Constructor:

Create an instance of the TreeGenerator using the TreeGenerator() constructor.

Required Object Specifications:

The TreeGenerator's makeTree function takes a JS object as an argument that stores the data you wish to represent with a tree. This object must follow a specific format illustrated in the example below. The object is recursive in nature to match the recursive structure of a tree.

            children: [] // An array of objects that follow the same structure that represents this node's child nodes. If empty, then this node is 
                            a leaf.

        }

Pyramid Generator:

Constructor:

Create an instance of the PyramidGenerator using the PyramidGenerator() constructor.

Required Object Specifications:

The PyramidGenerator's createPyramid function takes an array of objects as an argument. Each object in the array represents a section of the pyramid. The object at index 0 represents the top of the pyramid and the object at index array.length() - 1 represents the base of the pyramid. The object format is defined below:

Object Structure:

                    
        {
            value: '' // A string representing the text to appear inside a section of the pyramid,
            text: '' // A string representing the text to appear in a pop-up when a user hovers over this section (optional),
            color: '' // A string representing a valid CSS color value (hexcode, rgb, rgba) (optional, if blank a default color will be used),
        }

Example:

                    
        [
            {
                value: "Prestige",
                text: "Price: $30. Our most prestige package offers unlimited usage and our top speeds that will never slow down",
                color: '#d766ed'
            }, 
            {
                value: "Intermediate: $25",
                text: "The intermediate package offers 500GB monthly usage of high speed internet. (Speed may slow down during peak times)",
                color: '#eda166'
            }, 
            {
                value: "Basic: $15",
                text: "The basic package offers 100GB monthly usage of moderate speed internet. This package is suitable for those with minimal internet needs.",
                color: '#DAF7A6'
            }
        ]

VennDiagram Generator:

Constructor:

Create an instance of the VennDiagramGenerator using the VennDiagramGenerator() constructor.

Required Object Specifications:

The VennDiagramGenerator's makeVenn function takes an array of 3 objects as an argument. Each object in the array represents a section of the Venn Diagram. The object at index 0 represents the left section of the Venn Diagram, the object at index 1 represents the middle section (where similarities are stored), and the object at index 2 represents the right section of the Venn Diagram. The object format is defined below:

Object Structure:

                    
        {
            header: '' // A string representing the text to appear at the top of the Venn Diagram section,
            details: '' || [] // Either a string or an array of strings representing the product details of this section.
                                 If details is a string, a paragraph will be displayed in the section.
                                 If details is an array of strings, a bulleted list will be displayed in the section.
            color: '' // A string representing a valid CSS color value (hexcode, rgb, rgba) (optional, if blank a default color will be used),
        }

Example:

                    
        [
            {
                header: 'Product A: $30',
                details: `Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua. 
                Ut enim ad minim veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea commodo consequat. `
            },
            {
                header: 'Similarities:',
                details:  `Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua. 
                Ut enim ad minim veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea commodo consequat. `
                color: '#C85C5C'
            },
            {
                header: 'Product 2: $45',
                details: ['Lorem', 'ipsum', 'Dolor']
            }
        ]

Price Card Generator:

Constructor:

Create an instance of the PriceCardGenerator using the PriceCardGenerator() constructor.

Required Object Specifications:

The PriceCardGenerator's makeCard function takes a JS object as an argument. The object represents the data to to be displayed in the Price Card. The first 3 fields of the object are the name, price and frequency of the product. Every other key in the object is a string that will appear in the price card and the value of the key is a boolean that indicates whether there will be a checkmark or an 'X' appearing beside the string. Optionally, the object may include an 'img' key if an image is wanted in the price card. The object format is defined below:

Object Structure:

                    
        {
            name: '' // A string representing the name of the product/service,
            price: '' // A string representing the price of the product/service (include the currency symbol)
            frequency: '' // A string representing the frequency of the payment for this produce/service. 
                            (optional, if null no frequency will be displayed)
            'Any string': // The key can be any string and will appear directly inside the price card. The value must be a boolean in which true 
                             will be represented with a checkmark and false will be represented with an 'X'
                        
                          // Any number of user-defined keys can be included in the object.
            // img: '' A string that stores the link or path to an image. (optional, if null no image will be included in the price card)
        }

Example:

                    
        {
            name: 'Plan 3',
            price: '$20',
            frequency: null, 
            'Lorem ipsum dolor sit amet apple Lorem ipsum dolor sit amet': true,
            'Double': true,
            'Triple': false,
            'Power': true,
            img: 'https://t3.ftcdn.net/jpg/03/46/83/96/360_F_346839683_6nAPzbhpSkIpb8pmAwufkC7c5eD7wYws.jpg',
        }

Table Row Generator:

In the Table Row Generator, each product represents a row in the table.

Constructor:

Create an instance of the TableRowGenerator using the TableRowGenerator() constructor.

Required Object Specifications:

The TableRowGenerator's makeTable function takes an array of JS objects as an argument. Each object represents the data of a product to to be displayed as a row in the Table Matrix. The first 2 fields of each object are the name and price of the product. Every other key in the object is a string that will appear as a column header in the Table and the value of the key is either a boolean that indicates whether there will be a checkmark or an 'X' appearing in the cell for each category, or a regular string which will appear in the cell. Optionally, a 'text' field may be included in each object which represents the text which would appear in a pop-up when a user hovers over the name of each product. The object format is defined below:

NOTE: Each object in the array must have the exact same keys.

Object Structure:

                    
        {
            Name: ''        // A string representing the name of the product/service,
            Price: ''       // A string representing the price of the product/service,
            'Any String':   // The key is any string and will appear as a column header in the table matrix. The value can be either a boolean 
                               which is represented by a checkmark or 'X', or a regular string. The value will appear in the respective cell for
                               product.
                            // An object can have any number of strings for the column headers.
            text: ''        // A string representing the text to appear in a pop-up when a user hovers over this product's name 
                               (optional, if null no pop-up appears)
    
        },

Example:

                    
        [
            {
                Name: 'Product 1',
                Price: '$25',
                Bluetooth: true,
                Wireless: false,

            Name: ''        // A string representing the name of the product/service,
            Price: ''       // A string representing the price of the product/service,
            'Any String':   // The key is any string and will appear as a row header in the table matrix. The value can be either a boolean 
                               which is represented by a checkmark or 'X', or a regular string. The value will appear in the respective cell for
                               product.
                            // An object can have any number of strings for the row headers.
            text: ''        // A string representing the text to appear in a pop-up when a user hovers over this product's name 
                               (optional, if null no pop-up appears)
    
        },
